# Launch Docs

This folder tracks the Agent Computer V0 preview launch plan.

Agent Computer should be released as an experimental open-source preview, not as a stable OS, SaaS product, or finished marketplace. The launch goal is to help coding-agent power users understand the concept quickly, try one real workflow, and give feedback.

## Documents

- `launch-readiness-sprint.md`: the overall sprint plan and workstreams.
- `release-copy-spec.md`: what goes into the clean public release copy and what must be excluded.
- `release-checklist.md`: pre-GitHub checklist for V0 preview.
- `launch-strategy.md`: star-oriented launch strategy, positioning, demo, and channel plan.
- `launch-copy.md`: GitHub description, launch posts, Show HN body, demo script, and preview DM copy.
- `github-first-impression.md`: GitHub repo description, topics, social preview, README first-screen checklist, and starter issues.
- `agent-hub-plan.md`: the Agent Hub concept for sharing installable agent apps.

## Launch Principles

- Packaging should clarify the product, not simplify away agent behavior.
- README is for humans; `AGENTS.md` and agent app docs remain the runtime instructions.
- New users should mostly look at `workspace/inbox/` and `workspace/projects/`, while `computer/agents/`, `computer/tools/`, `computer/system/`, and `computer/templates/` are the operating layer.
- The first public claim is "v0 preview", not "stable product".
- No private memory, real contacts, private reports, converted private documents, or generated user artifacts belong in the release copy.
