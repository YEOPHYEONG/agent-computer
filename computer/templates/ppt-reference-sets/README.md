# PPT Reference Sets

Open, repo-native visual reference boards for `ppt-builder`.

These are not copied from third-party decks. They are lightweight original SVG boards that define layout mood, density, component grammar, and color direction. If the user provides reference images, use those first. Otherwise choose one of these sets and translate it into editable PPT text, shapes, tables, charts, and diagrams.

| Set | Use When | Board |
|---|---|---|
| `consulting-report` | Strategy, market, operational, or recommendation decks need crisp evidence and executive clarity. | `consulting-report/reference-board.svg` |
| `research-briefing` | Research-heavy outputs need source trust, question ledgers, evidence maps, and careful caveats. | `research-briefing/reference-board.svg` |
| `startup-strategy` | Early product, launch, open-source, or GTM narratives need momentum plus proof. | `startup-strategy/reference-board.svg` |
| `product-deck` | Product/feature/solution decks need flows, screens, benefits, constraints, and roadmap structure. | `product-deck/reference-board.svg` |
| `executive-board` | Board/executive updates need dense but calm KPI, risk, decision, and next-action slides. | `executive-board/reference-board.svg` |
| `editorial-visual` | Brand, thought leadership, or narrative reports need richer visual rhythm and editorial pacing. | `editorial-visual/reference-board.svg` |

## How To Use

1. Select the closest reference set in the PPT content/design specs.
2. State why it fits the source and audience.
3. Translate its layout grammar into editable PPT elements.
4. Do not paste the SVG board into the final deck as a full-slide image.
5. If the final deck uses images, limit them to hero visuals, product/place/person images, mockup interiors, or carefully bounded texture/asset regions.
