# PPT QA Checklist

## Mode

- [ ] Premium workflow used: content spec -> design spec -> build plan -> prototype/render QA -> editable PPTX reconstruction.
- [ ] No generic PPTX was created before the prototype/sample-slide gate.

## Planning

- [ ] Content spec exists and maps source sections to slide roles or source/reference treatment.
- [ ] Design spec exists and assigns layout roles.
- [ ] Build plan exists and identifies editable elements.
- [ ] A user-provided or built-in reference set was selected before visual planning.
- [ ] The design spec explains how the reference set was translated into this deck.
- [ ] Sample slides were reviewed first when the deck was substantial.

## Editability

- [ ] Titles are editable text.
- [ ] Body text is editable text.
- [ ] Tables are editable where possible.
- [ ] Charts are native or editable where possible.
- [ ] Cards and diagrams are shapes.

## Content Fidelity

- [ ] Important source meaning, logic, evidence, caveats, and decisions were preserved.
- [ ] Dense content was condensed into designed proof objects or split only when needed.
- [ ] Source-heavy detail is traceable through the source map or appendix when needed.
- [ ] Speaker notes, appendix/source map, and omissions are separated from slide-body content.
- [ ] No claim, caveat, material evidence item, or action item disappeared without approval.
- [ ] Repeated wording, local paths, tool logs, and source transcripts were not forced into slide bodies by default.

## Layout

- [ ] No text overflow.
- [ ] No incoherent overlap.
- [ ] Visual hierarchy is clear.
- [ ] Charts and tables are readable.
- [ ] Each main slide has one key message.
- [ ] Layout roles are varied enough for the deck length.
- [ ] Main narrative slides are not plain text-only dumps.

## Visual Prototype

- [ ] HTML/prototype or equivalent rendered previews exist for premium mode.
- [ ] Preview PNGs or contact sheet were inspected.
- [ ] HTML/screenshots were not inserted as full-slide images in the final PPTX.

## Final

- [ ] PPTX opens correctly.
- [ ] Rendered previews match the intended design.
- [ ] Remaining issues are documented.
