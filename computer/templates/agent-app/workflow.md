# Workflow

1. Read `AGENTS.md`.
2. Apply the Agent Computer Boundary Rule: use workspace-native tools/files/memory before external apps.
3. Identify the requested output and active project.
4. Execute with the agent's tools or templates.
5. Save durable output under `projects/<project-slug>/<work-type>/`.
6. Add approval gates for any external app/account or irreversible action.
