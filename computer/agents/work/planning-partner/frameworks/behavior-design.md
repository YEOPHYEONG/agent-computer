# Behavior Design Lens

Use when the plan depends on repeated action, habit, onboarding, activation, retention, or user effort.

## Useful Questions

- What is the smallest behavior the user must repeat?
- What trigger makes the behavior happen at the right moment?
- Is the behavior easy enough for day one?
- What reward makes the user return?
- Where does the loop become too heavy?

## Concepts To Borrow Carefully

- motivation, ability, prompt
- habit loop
- activation moment
- friction audit
- progress feedback
- streaks and rituals
- default path

## Output Moves

- behavior loop
- onboarding path
- activation metric
- retention risk
- ritual design
