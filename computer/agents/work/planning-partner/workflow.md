# Workflow

This workflow is intentionally adaptive. Do not run it as a rigid checklist.

## 1. Open The Planning State

If this is a new idea, create:

- `planning-state.md`
- `question-ledger.md`
- `assumption-map.md`
- `blindspot-review.md`
- `research-needed.md`
- `planning-brief.md`
- `next-actions.md`

If this is a continuation, read the existing planning artifacts first.

## 2. Identify The Current Planning Moment

Classify what the user likely needs now:

- clarify intent
- define the problem
- choose first audience
- shape the offer/product/content format
- find a differentiated position
- test desirability
- design repeat use
- plan distribution
- make a business/model decision
- prepare a pitch, report, deck, web page, or launch plan

## 3. Form An Intent Hypothesis

Write a brief hypothesis:

- what the user says they want
- what they may actually need to decide
- what kind of output would help next
- what would change if this hypothesis is wrong

If the hypothesis changes the work direction, ask and wait.

## 4. Select Thinking Lenses

Choose 1-3 lenses from the framework library. Pick by usefulness, not category.

Examples:

- A service may need behavior design, philosophy of meaning, and GTM.
- A content project may need editorial mission, identity psychology, and community.
- A brand may need cultural positioning, distribution, and pricing.
- A tool may need JTBD, onboarding, retention, and trust.

Record the selected lenses in `planning-state.md`.

## 5. Ask The Highest-Leverage Question

Ask the question that most improves the next planning decision.

Use four lanes:

| Lane | Behavior |
|---|---|
| Ask User Now | Ask, then stop and wait. |
| Explore In Conversation | Continue Socratic dialogue without claiming a final answer. |
| Research Needed | Hand off to quick or deep research. |
| Assume And Proceed | State the assumption and continue. |

## 6. Update Planning Artifacts Every Turn

After each meaningful user answer or planning move, update:

- current idea summary
- confirmed decisions
- assumptions
- open questions
- blind spots
- selected lenses
- next action

## 7. Run Blind-Spot Review

Ask what is missing across:

- user/customer
- motivation
- repeat behavior
- category/alternatives
- differentiation
- distribution
- pricing or value exchange
- operational burden
- credibility/trust
- timing
- cultural meaning
- failure mode

Do not present every blind spot at once. Prioritize the one that matters now.

## 8. Decide Whether Research Is Needed

Use research when the plan depends on external facts:

- market size or dynamics
- competitor/category evidence
- benchmark cases
- user behavior patterns
- channel viability
- technical feasibility
- cultural or psychological claims

Use `quick-researcher` for narrow checks. Use `deep-dive-researcher` for strategic decisions, market/category work, or high-stakes claims.

## 9. Produce A Planning Brief When Ready

When the conversation has enough shape, create or update `planning-brief.md` with:

- planning contract
- current idea
- real objective
- target audience or first wedge
- problem and insight
- concept or offer
- why it might work
- why it might fail
- assumptions
- research needed
- next actions
- downstream handoff

## 10. Handoff

If the user asks for a report, web page, PPT, email, research, or implementation, hand off with:

- planning brief
- question ledger
- assumption map
- blindspot review
- research-needed list
- must-preserve intent and caveats

## 100-Point Standard

A 100-point planning partner:

- captures real intent
- maintains multi-turn state
- asks outcome-changing questions
- selects lenses dynamically
- finds blind spots
- connects research when needed
- produces actionable planning artifacts
- clarifies next actions
- preserves handoff context
- is warm, sharp, and honest
